# goMFP - Go MultiFunPlayer 제어 센터

> [!NOTE]
> 이 프로젝트는 C#으로 작성된 원본 MultiFunPlayer 프로젝트([Yoooi0/MultiFunPlayer](https://github.com/Yoooi0/MultiFunPlayer))를 기반으로 Go 언어로 재구현한 버전입니다.

**goMFP**는 Go 언어로 개발된 고성능 미디어 동기화 및 햅틱 디바이스 제어 소프트웨어(MultiFunPlayer 제어 센터)입니다. Fyne v2 기반의 미려하고 반응성이 뛰어난 데스크톱 GUI 환경과 함께, 모바일 및 원격 디바이스에서도 간편하게 접속하여 실시간 모니터링과 튜닝이 가능한 모던 웹 인터페이스(WebUI)를 동시에 제공합니다.

## 🌟 주요 특징 (Key Features)

### 📊 호환성 및 테스트 현황 (Compatibility & Status)

### 1. 강력한 미디어 플레이어 연동
* **다양한 소스 지원**: 내장 플레이어(Internal), MPC-HC, MPV(Named Pipe), VLC(HTTP API), DeoVR, HereSphere 등 다양한 VR 및 2D 미디어 소스와 원활히 동기화됩니다.
* **시간 동기화**: 프레임 단위의 초정밀 타임스탬프 동기화를 통해 디바이스 모션 딜레이를 최소화합니다.

### 2. 프리미엄 다축 히트맵 시각화 및 인터랙티브 제어
* **안티에일리어싱(AA) 곡선**: Fyne GUI에 Xiaolin Wu 선 그리기 알고리즘을 구현하여 지저분한 계단 현상 없이 극도로 부드럽고 수려한 곡선을 출력합니다.
* **실시간 글로우 효과**: 현재 재생 시간에 해당하는 L0 축 위치에 빨간색 글로우 원(Glow Circle)이 동적으로 애니메이션 처리되어 직관적인 피드백을 줍니다.
* **다축 오버레이**: L0 축을 기반으로 하되, 로드된 다른 서브 축들(L1~A2)을 반투명(투명도 30%)하게 그 위에 겹쳐 렌더링함으로써 전체 모션의 흐름을 한눈에 조망할 수 있습니다.
* **더블 버퍼링(Double-Buffering)**: 미디어 교체나 스크립트 로드 시에만 백그라운드 이미지 캐시를 갱신하는 설계를 적용해 CPU 연산 낭비를 0%로 줄이고 깜빡임(Flicker)을 원천 차단했습니다.
* **클릭 이동(Click-to-Seek)**: Fyne GUI의 히트맵 위젯 및 WebUI의 캔버스 영역 중 원하는 위치를 클릭하면 즉시 동영상 재생 위치로 고속 이동(Seek)합니다.

### 3. 유연한 출력 채널 관리
* UDP, TCP, WebSockets, Serial(Port 직접 입력 및 활성 포트 드롭다운 스캔 선택 지원), File 쓰기 등 다양한 경로로 TCode 프로토콜 출력을 전송할 수 있습니다.
* 각 채널별로 전송 대상 주소, 보오레이트(Baud Rate), 자동 연결 유무를 개별 설정할 수 있습니다.

## 🚀 실행 방법 (Usage)

### 기본 실행
다운로드한 `goMFP.exe` 파일을 더블클릭하여 GUI 모드로 즉시 실행합니다. (기본 실행 시에는 로그 파일이 생성되지 않아 폴더를 깨끗하게 유지합니다.)

### 디버그 및 로그 수집 모드
시작 실패 오류나 플레이어 연결 예외를 기록/디버깅하려면 명령 프롬프트(CMD) 또는 파워셸(PowerShell)에서 아래의 실행 옵션 플래그 중 하나를 주어 실행합니다:
```powershell
# 옵션을 사용하여 실행하면 실시간으로 goMFP_startup.log 파일이 실행 경로에 생성됩니다.
.\goMFP.exe --log

# 단축 옵션도 사용 가능합니다.
.\goMFP.exe -d
```

## 🛠️ 빌드 방법 (How to Build)

### 요구사항
1. **Go Compiler**: Go 1.20 버전 이상 설치 권장.
2. **GCC 툴체인**: Fyne GUI 컴파일을 위해 Windows 기준 MinGW-w64 툴체인이 필요합니다.

### 컴파일 명령어
명령 프롬프트(CMD)나 파워셸(PowerShell)에서 아래 명령어를 실행하여 Windows 백그라운드 전용(콘솔창이 뜨지 않는 GUI 모드) 실행 파일을 빌드합니다:
```powershell
# Cgo를 통한 컴파일을 위해 GCC 경로가 환경 변수 PATH에 존재해야 합니다.
go build -ldflags "-s -w -H=windowsgui" -o goMFP.exe main.go
```

## 📂 프로젝트 구조 (Project Structure)
* `main.go`: Fyne GUI 레이아웃, 더블 버퍼링 히트맵 드로잉 및 애플리케이션의 핵심 진입점.
* `pkg/`:
  * `player/`: 미디어 소스 모니터링 및 스크립트 키프레임 파싱 코디네이터.
  * `server/`: REST API 및 WebSocket 통신을 담당하는 웹 서버 패키지.
  * `settings/`: JSON 기반의 설정을 관리하는 설정 매니저 모듈.
* `web/`: WebUI 정적 아셋 (HTML, CSS, JS) 파일이 포함된 웹 서버 루트.
