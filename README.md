# goMFP - Go MultiFunPlayer Control Center

[한국어 버전 README 바로가기 (Go to Korean Version README)](README_kr.md)

> [!NOTE]
> This project is a Go-based implementation inspired by the original C# MultiFunPlayer project: [Yoooi0/MultiFunPlayer](https://github.com/Yoooi0/MultiFunPlayer).

**goMFP** is a high-performance media synchronization and haptic device control software (MultiFunPlayer Control Center) developed in Go. It offers a beautiful, responsive desktop GUI environment based on Fyne v2, alongside a modern web interface (WebUI) for easy access, real-time monitoring, and tuning from mobile and remote devices.

### 1. Robust Media Player Integration
* **Wide Range of Supported Players**: Seamlessly syncs with various VR and 2D media sources including Internal Player, MPC-HC, MPV (Named Pipe), VLC (HTTP API), DeoVR, and HereSphere.
* **Time Sync**: Minimizes device motion delay through ultra-precise frame-level timestamp synchronization.

### 2. Premium Multi-Axis Heatmap Visualization & Interactive Control
* **Anti-Aliased (AA) Curves**: Implements Xiaolin Wu's line drawing algorithm in the Fyne GUI to render extremely smooth, vector-like curves without blocky aliasing artifacts.
* **Real-time Glow Effect**: Features a red glow circle animating dynamically at the current playback position on the L0 axis for intuitive real-time feedback.
* **Multi-Axis Overlay**: Renders the loaded secondary axes (L1 to A2) on top of the L0 base curve with translucency (30% opacity), allowing you to oversee the entire motion pattern at a glance.
* **Double-Buffering**: Applies a double-buffered background image cache that regenerates only when the video changes or active scripts are toggled, reducing CPU overhead to 0% and completely eliminating drawing flicker.
* **Click-to-Seek**: Instantly jumps to the clicked playback position by clicking anywhere on the Fyne GUI heatmap widget or the WebUI canvas.

### 3. Flexible Output Channel Management
* Sends TCode protocol streams through UDP, TCP, WebSockets, Serial (supports manual port entry or scanning with dropdown selection), or writes directly to a File.
* Allows individual channel configurations for target address/port, baud rate, and auto-connect behavior.

## 🚀 Usage

### Basic Execution
Double-click the downloaded `goMFP.exe` to launch the GUI mode immediately. (No log files are generated by default, keeping your folder clean.)

### Debug & Log Collection Mode
To log and debug startup failures or player connection issues, launch from a command prompt (CMD) or PowerShell with one of the following logging flags:
```powershell
# Running with the flag generates the goMFP_startup.log file in the execution directory.
.\goMFP.exe --log

# Short flag is also supported
.\goMFP.exe -d
```

## 🛠️ How to Build

### Requirements
1. **Go Compiler**: Go version 1.20 or newer recommended.
2. **GCC Toolchain**: MinGW-w64 toolchain is required on Windows to compile Fyne GUI.

### Build Command
Run the following command in CMD or PowerShell to build the background GUI executable (hiding the command console window):
```powershell
# GCC must be added to your PATH environment variable for Cgo compilation.
go build -ldflags "-s -w -H=windowsgui" -o goMFP.exe main.go
```

## 📂 Project Structure
* `main.go`: Entry point containing Fyne GUI layout, double-buffered heatmap drawing, and key app logic.
* `pkg/`:
  * `player/`: Coordinator managing media source monitoring and script keyframe parsing.
  * `server/`: Web server package handling REST API and WebSocket communication.
  * `settings/`: Settings manager module governing JSON configuration files.
* `web/`: Root directory of the web server containing WebUI static assets (HTML, CSS, JS).
